# Day 27 – GitHub Profile Makeover
## Build My Developer Identity

Today was not a coding day — it was a branding and positioning day.

I treated my GitHub profile as my developer resume and optimized it to clearly represent my DevOps journey.

---

## 🔍 Task 1: Profile Audit (Before Changes)

When I visited my profile as a stranger, I noticed:

- My profile did not clearly communicate that I am focused on DevOps.
- Some repositories lacked proper descriptions.
- Repository organization was not structured by domain.
- Pinned repositories did not strongly reflect my DevOps direction.
- My learning journey (90 Days of DevOps) was not highlighted properly.

From a recruiter’s perspective, my focus was not immediately clear.

Here is how my Profile look like it was before.

---

## 🛠 Task 2: Created Profile README

I created a special repository with my GitHub username to activate a profile README.

### What I Included:

- Clear introduction: DevOps Engineer in Progress
- Mentioned 90 Days of DevOps challenge
- Listed tools I’ve worked with:
  - Linux (Ubuntu)
  - Git & GitHub
  - Docker
  - Bash / Shell Scripting
  - Python
- Mentioned upcoming learning goals:
  - CI/CD
  - Kubernetes
  - Cloud (AWS)
  - Infrastructure as Code
- Added contact details (LinkedIn / Email)

Goal: Anyone landing on my profile should immediately understand my direction.

---

## 📂 Task 3: Repository Organization

### 1️⃣ 90-days-of-devops

- Structured folders by day
- Added clear README explaining the challenge
- Ensured consistent naming format

### 2️⃣ shell-scripts

- Moved Days 16–21 scripts
- Added README explaining what each script does
- Added proper .gitignore

### 3️⃣ python-scripts

- Moved Days 7–15 Python files
- Documented each script
- Organized cleanly

### 4️⃣ devops-notes

- Added:
  - git-commands.md
  - Shell scripting cheat sheet
- Organized notes by:
  - Linux
  - Git
  - Python
  - Docker

All repositories now:
- Use hyphen naming (no spaces)
- Have proper descriptions
- Contain structured README.md
- Do not expose secrets

## 📌 Task 4: Pinned Repositories

Pinned repositories now represent my DevOps focus:
